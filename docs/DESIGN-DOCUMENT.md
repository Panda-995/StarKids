# 🌟 StarKids - 小朋友奖励乐园 · 全面设计文档

> 版本: v1.0 | 日期: 2026-04-30 | 目标用户: 3-12 岁儿童及其家长
> 定位: 家庭奖励机制管理平台 — 让孩子爱上劳动与成长

---

## 一、产品理念

### 1.1 核心概念

> **用游戏化的方式，让好习惯自然生长**

StarKids 是一个面向家庭的儿童奖励管理系统，通过「任务 → 积分 → 兑换」的正向反馈循环，帮助孩子建立责任感、延迟满足能力和目标管理意识。

### 1.2 设计原则

| 原则 | 说明 |
|---|---|
| **童趣优先** | 界面必须可爱、明亮、充满惊喜，让孩子第一眼就爱上 |
| **家长可控** | 所有规则由家长设定，孩子只负责参与 |
| **正向激励** | 不惩罚、只奖励，每次互动都带来快乐 |
| **成长可见** | 进度、成就、变化可视化，增强获得感 |
| **安全无害** | 无社交功能、无外部内容、数据本地化 |

## 三、核心功能模块

### 3.1 任务系统 ⭐

#### 3.1.1 任务类型

| 类型 | 图标 | 说明 | 示例 |
|---|---|---|---|
| **日常任务** | 🔄 | 可每日/每周重复 | 收拾玩具、写作业、刷牙 |
| **一次性任务** | 🎯 | 完成后不再生成 | 整理书架、大扫除 |
| **挑战任务** | 🏆 | 高难度、高积分 | 独立完成一个手工作品 |
| **习惯养成** | 🌱 | 连续打卡类 | 连续7天阅读打卡 |

#### 3.1.2 任务属性

```yaml
任务:
  id: UUID
  名称: "收拾自己的房间"
  描述: "把玩具放回箱子，被子铺平整"    # 小朋友能看懂的描述
  积分: 10                              # 完成任务获得的积分
  类型: 日常/一次性/挑战/习惯           # 任务分类
  频率: 每日/每周/手动                   # 生成频率
  目标年龄段: 3-12                       # 适用年龄
  难度: ⭐/⭐⭐/⭐⭐⭐                    # 难度等级
  图片: URL                             # 任务配图
  完成条件:                              # 多种验证方式
    - 家长审核（默认）                   # 完成后需家长批准
    - 自动通过（简单任务）               # 如"刷牙"类
    - 定时触发（习惯类）                 # 连续打卡
  有效期: 2026-05-01 ~ 2026-06-30       # 可选
  适用对象: 所有 / 指定小朋友             # 谁可以做
  每日次数限制: 1                        # 每日可完成的次数
  状态: 启用/停用
  创建时间: DateTime
  创建人: 家长A
```

### 3.2 积分系统 🌟

#### 3.2.1 积分规则

| 功能 | 说明 |
|---|---|
| **基础积分** | 完成任务获得对应积分 |
| **连续奖励** | 连续 N 天完成所有任务，额外 +Bonus |
| **超额奖励** | 超出目标完成次数，获得额外积分 |
| **周末双倍** | 周末完成任务积分 × 2（可选开关） |
| **特殊节日** | 生日当天积分 × 3（自动触发） |
| **积分扣除** | 家长可手动扣除（用于不当行为，谨慎使用） |
| **积分上限** | 可设置每日积分上限（防止过度） |

### 3.5 家庭宠物 🐱

#### 3.5.1 概念

每个小朋友拥有一只虚拟宠物，宠物会随着任务完成而成长！这是**我特别推荐的增强功能**——它能把枯燥的任务变成一场养成游戏。

#### 3.5.2 宠物成长规则

| 成长值 | 宠物状态 | 表现 |
|---|---|---|
| 0-50 | 🥚 蛋宝宝 | 一颗可爱的蛋在摇晃 |
| 50-200 | 🐣 破壳啦 | 小宠物刚出生，小小的 |
| 200-500 | 🐥 成长中 | 慢慢变大，开始有表情 |
| 500-1000 | 🦋 进阶 | 进化成更帅气的形态 |
| 1000+ | 👑 完全体 | 终极形态，闪闪发光！ |

#### 3.5.3 宠物与任务的关联

```
完成任务 → 宠物获得成长值 → 升级进化
  ↓
宠物心情值: 完成任务开心, 超过3天没做任务会"难过"
  ↓
宠物可以换装: 用积分给宠物买小衣服/小帽子
```

## 六、数据模型设计

```prisma
// ═══════════════════════════════════════
//  用户与家庭
// ═══════════════════════════════════════

model Family {
  id          String    @id @default(uuid())
  name        String               // "朵朵一家"
  avatar      String?              // 家庭头像
  timezone    String    @default("Asia/Shanghai")
  members     FamilyMember[]
  tasks       Task[]
  rewards     Reward[]
  pets        Pet[]
  createdAt   DateTime @default(now())
}

model FamilyMember {
  id         String    @id @default(uuid())
  familyId   String
  family     Family    @relation(fields: [familyId], references: [id])
  user       User      @relation(fields: [userId], references: [id])
  userId     String
  role       MemberRole            // PARENT / KID
  nickname   String?               // 小朋友的昵称
  avatar     String?               // 头像
  age        Int?
  points     Int       @default(0) // 当前积分
  createdAt  DateTime @default(now())

  tasks      Task[]      @relation("TaskAssignees")
  completions TaskCompletion[]
  redemptions RewardRedemption[]
  pointLogs  PointLog[]
  pet        Pet?
  comments   Comment[]
}

model User {
  id            String    @id @default(uuid())
  email         String    @unique
  name          String?
  passwordHash  String?
  role          UserRole  @default(PARENT)
  familyMembers FamilyMember[]
  createdAt     DateTime  @default(now())
}

enum UserRole {
  PARENT
  KID
}

enum MemberRole {
  PARENT
  KID
}

// ═══════════════════════════════════════
//  任务
// ═══════════════════════════════════════

model Task {
  id            String      @id @default(uuid())
  familyId      String
  family        Family      @relation(fields: [familyId], references: [id])
  name          String                // "刷牙"
  description   String?               // 给小朋友看的描述
  type          TaskType              // DAILY / ONETIME / CHALLENGE / HABIT
  frequency     TaskFrequency @default(MANUAL) // DAILY / WEEKLY / MANUAL
  points        Int                   // 完成获得的积分
  difficulty    Difficulty  @default(MEDIUM)   // EASY / MEDIUM / HARD
  image         String?               // 任务配图
  category      String?               // 家务/学习/运动/习惯/其他
  autoApprove   Boolean     @default(false)   // 是否自动通过
  maxDaily      Int         @default(1)       // 每日最多完成次数
  startDate     DateTime?             // 有效期开始
  endDate       DateTime?             // 有效期结束
  status        TaskStatus  @default(ACTIVE)  // ACTIVE / INACTIVE
  sortOrder     Int         @default(0)
  creatorId     String
  assignees     FamilyMember[] @relation("TaskAssignees")
```

## 十、我额外推荐的功能 ✨

作为一个给小朋友的奖励系统，除了你提到的功能外，强烈建议加上以下功能：

### 10.1 🐱 虚拟宠物养成 (强烈推荐！)

让小朋友完成任务来养一只电子宠物，宠物会随任务完成而成长进化。这是**增强粘性的核心功能**，能把"要我干"变成"我要干"。

### 10.2 🎵 音效系统

每个操作配上可爱的音效——任务完成的"叮"、积分到账的星星声、兑换成功的礼花声。声音反馈对小朋友的激励效果远超视觉。

### 10.3 📝 家长鼓励留言

任务审核通过时，家长可以留一句鼓励的话。比如"宝宝今天真棒！妈妈为你骄傲 ❤️"——这比积分更有温度。

### 10.4 🎨 主题皮肤

让小朋友可以解锁不同的主题皮肤（海底世界、太空探险、森林王国），用积分解锁。增加趣味性。

### 10.5 📊 周报/月报自动推送

每周自动生成小朋友的成长报告推送给家长，包含任务完成率、积分趋势、成就获取等。让成长看得见。

### 10.6 🔔 智能提醒

- 每日任务提醒："朵朵，今天的任务还没完成哦~小狐狸在等你！"
- 连续打卡提醒："再坚持2天就有连续奖励啦！"
- 宠物想念提醒："你的小猫想你了，快去完成任务陪它吧~"

### 10.7 🎯 任务模板库

预置丰富的任务模板，家长可直接使用：
- **家务类**: 收拾房间、洗碗、倒垃圾、整理书桌
- **学习类**: 写作业、阅读、练字、背单词
- **运动类**: 跳绳、跑步、打球、做操
- **习惯类**: 刷牙、洗脸、早睡早起、不挑食
- **社交类**: 说谢谢、分享玩具、主动打招呼

### 10.8 🎁 神秘盲盒

家长可以设置"神秘奖励"——小朋友用积分可以开盲盒，随机获得不同奖励。小朋友最爱惊喜了！

### 10.9 👨‍👩‍👧‍👦 多孩友好排行

多孩家庭的温和竞争机制，但强调"一起进步"而非"互相比较"。

### 10.10 💾 数据导出

支持导出 CSV/JSON 格式的数据，方便家长备份或在其他平台使用。
